## Лаба 1. Алгоритм Дейкстры

### Условие
Своя реализация алгоритма Дейкстры для поиска кратчайших путей от заданной вершины до всех остальных. NetworkX разрешается использовать для представления графа и визуализации, но **не** для самого алгоритма.

### Реализация
Граф принимается в двух форматах — как **матрица смежности** (нули = отсутствие ребра) и как **словарь смежности** (`dict[node -> list[(neighbor, weight)]]`). Внутренний хелпер `_to_adjacency` приводит оба к одному виду.

Приоритетная очередь — `heapq` (бинарная min-куча). Устаревшие записи в куче не удаляются явно: при извлечении проверяется `d > dist[u]` и они просто пропускаются. Это даёт стандартную сложность `O((V + E) log V)`.

Параллельно с `dist` ведётся словарь предшественников `prev`, по которому хелпер `reconstruct_path` восстанавливает сам путь.

### Проверка
Результат на тестовом графе из 5 вершин сверяется с `nx.single_source_dijkstra_path_length` — расхождений быть не должно (`assert ref == ours`).

Визуализация рисует граф через `nx.spring_layout` и подсвечивает один из найденных кратчайших путей красным.

## Лаба 2. Центральность по собственному вектору

### Условие
Подобрать **вручную** граф из 50 узлов так, чтобы значения центральности `c(i)`, упорядоченные по индексу вершины, имели форму «спуск → яма → подъём → пик → спуск» (примерно как кубический полином).

### Решение в двух частях

**1. Веб-редактор `lab2_ui.html`.** Самодостаточный HTML — открывается в браузере без сервера. Возможности:

- 50 узлов, ЛКМ + ЛКМ переключает ребро, drag двигает узел, ПКМ снимает все рёбра вершины
- Зум колесом мыши, панорама через Space+drag
- Раскладки: круг / сетка / линия / случайно
- Пресеты структур: цепочка, цикл, звезда, гантель, двудольный, и **«Кубик»** — две клики K₁₂ на индексах [0..11] и [20..31] плюс перемычки (4,24) и (5,25), которые снимают вырождение собств. значений и дают форму ↓pit↑peak↓
- В реальном времени пересчитывается центральность и рисуется график `c(i)` по индексу
- Сохранение и загрузка графа в JSON
- Предупреждение, если граф разорван на несколько компонент

**2. Скрипт `lab2_analysis.py`.** Берёт сохранённый `lab2_graph.json`, строит `nx.Graph`, считает `nx.eigenvector_centrality_numpy`, рисует итоговую кривую `c(i)` и раскрашенный по центральности граф для отчёта.

### Особенности

**JS-реализация центральности использует сдвинутый степенной метод** — итерацию `(A + d_max · I) · x` вместо обычного `A · x`. Без сдвига на двудольных графах (пути, чётные циклы) есть пара собств. значений `±λ_max`, и итерация осциллирует. После сдвига все собств. значения положительны, итерация сходится. Расхождение с `nx.eigenvector_centrality_numpy` на связных графах — порядка `10⁻¹²`.

**Граф должен быть связным.** `nx.eigenvector_centrality_numpy` на разорванных графах кидает `AmbiguousSolution`. В скрипт добавлен `assert nx.is_connected(G)` с понятным сообщением, в редакторе — красная плашка-предупреждение.

## Лаба 3. Модель Эрдёша–Реньи

### Условие
Сгенерировать граф `G(n, p)` с `n = 60`, `p = 0.45`. Посчитать эмпирическую среднюю степень и сравнить с теоретическим значением из лекций.

### Теория

| Величина             | Формула              | При наших параметрах |
| -------------------- | -------------------- | -------------------- |
| Среднее число рёбер  | `m = n(n-1)/2 · p`   | `796.5`              |
| Средняя степень      | `c = (n-1) · p`      | `26.55`              |
| Распределение степеней | стремится к Пуассону `P(x) = c^x e^(-c) / x!` | — |

### Эксперимент

1. Одна реализация графа: считаем число рёбер и среднюю степень. На одной выборке отклонение от теоретического значения вполне ожидаемо.
2. Усреднение по 500 независимым генерациям: среднее эмпирической степени почти точно совпадает с `(n-1)·p`, стандартное отклонение по выборкам ≈ `0.69`.
3. Гистограмма распределения степеней (по всем 500 графам) сравнивается с пуассоновским распределением с `c = 26.55`. Качественно похоже, но не идеально — формально предельный переход «биномиальное → Пуассон» работает при `n → ∞, np = const`, чего здесь нет.

### Типичный вывод

```
Параметры: n=60, p=0.45

Рёбер (эмпирически):     771
Рёбер (теоретически):    796.50

Средняя степень эмпир.:  25.7000
Средняя степень теор.:   26.5500   = (n-1) * p
Расхождение:             0.8500

Среднее по 500 генерациям: 26.5719
Теория:                    26.5500
Std по генерациям:         0.6941
```
