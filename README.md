## Задание 1. Алгоритм Дейкстры

### Условие

Своя реализация алгоритма Дейкстры для поиска кратчайших путей от заданной вершины до всех остальных. NetworkX разрешается использовать для представления графа и визуализации, но **не** для самого алгоритма.

### Представление графа

Принимаются два формата.

Матрица смежности - список списков, нули означают «ребра нет»:

```python
matrix = [
    [ 0, 7, 9, 0, 0],
    [ 7, 0,10,15, 0],
    [ 9,10, 0,11, 2],
    [ 0,15,11, 0, 6],
    [ 0, 0, 2, 6, 0],
]
```

Словарь смежности - `dict[node -> list[(neighbor, weight)]]`:

```python
adj = {
    0: [(1, 7), (2, 9)],
    1: [(0, 7), (2, 10), (3, 15)],
    2: [(0, 9), (1, 10), (3, 11), (4, 2)],
    3: [(1, 15), (2, 11), (4, 6)],
    4: [(2, 2), (3, 6)],
}
```

Внутренний хелпер `_to_adjacency` приводит оба формата к одному виду.

### Алгоритм

1. Инициализация: `dist[start] = 0`, для остальных вершин `+inf`. В мин-кучу кладём `(0, start)`.
2. Извлекаем из кучи пару `(d, u)` с минимальной текущей дистанцией.
3. Если `d > dist[u]` - запись устарела (мы уже нашли путь короче), пропускаем. Ленивое удаление без явного перестроения кучи.
4. Иначе релаксируем исходящие рёбра `u → v`: если `dist[u] + w(u,v) < dist[v]`, обновляем `dist[v]`, ставим `prev[v] = u` и кладём новую запись `(nd, v)` в кучу.
5. Куча пустая => `dist` содержит длины кратчайших путей. По `prev` восстанавливаем сами пути через `reconstruct_path`.

### Сложность

`O((V + E) log V)` за счёт бинарной кучи `heapq`. Отрицательные веса отклоняются `ValueError`

### Проверка

Результат на тестовом графе из 5 вершин сверяется с `nx.single_source_dijkstra_path_length`

Визуализация рисует граф через `nx.spring_layout` и подсвечивает один из найденных кратчайших путей красным.

### Пример вывода

```
Длины кратчайших путей из вершины 0:
  0 -> 0: 0    путь: [0]
  0 -> 1: 7    путь: [0, 1]
  0 -> 2: 9    путь: [0, 2]
  0 -> 3: 17   путь: [0, 2, 4, 3]
  0 -> 4: 11   путь: [0, 2, 4]

Сверка с NetworkX:
  NetworkX: {0: 0, 1: 7, 2: 9, 3: 17, 4: 11}
  Наш:      {0: 0, 1: 7, 2: 9, 3: 17, 4: 11}
```

## Задание 2. Центральность по собственному вектору

### Условие

Подобрать **вручную** граф из 50 узлов так, чтобы значения центральности `c(i)`, упорядоченные по индексу вершины, имели форму «спуск → яма → подъём → пик → спуск» (примерно как кубический полином).

### Теоретическая справка

Центральность по собственному вектору - это решение уравнения `A·c = λ·c`, где `A` - матрица смежности, `λ` - наибольшее собственное значение. По смыслу: значение `c[i]` пропорционально сумме `c[j]` всех соседей. Вершина важна, если её соседи тоже важны.

`nx.eigenvector_centrality_numpy` находит главный собственный вектор через `scipy.sparse.linalg.eigs` и нормирует так, что `||c||₂ = 1`. 

### Решение в двух частях

**1. Веб-редактор `lab2_ui.html`.** Открывается в браузере без сервера. Возможности:

- 50 узлов, ЛКМ + ЛКМ переключает ребро, drag двигает узел, ПКМ снимает все рёбра вершины
- В реальном времени пересчитывается центральность и рисуется график `c(i)` по индексу
- Сохранение и загрузка графа в JSON

**2. Скрипт `lab2_analysis.py`.** Берёт сохранённый `lab2_graph.json`, строит `nx.Graph`, считает `nx.eigenvector_centrality_numpy`, рисует итоговую кривую `c(i)` и раскрашенный по центральности граф.

```python
with open('lab2_graph.json', encoding='utf-8') as f:
    data = json.load(f)

G = nx.Graph()
G.add_nodes_from(range(data['n']))
G.add_edges_from(data['edges'])

assert nx.is_connected(G), 'граф разорван'

centrality = nx.eigenvector_centrality_numpy(G)
c = np.array([centrality[i] for i in range(data['n'])])
```

### Результат

На итоговом графике `c(i)` по индексу должна быть видна заданная форма «спуск → яма → подъём → пик → спуск». Сам граф визуализируется в координатах из редактора, размер и цвет узлов масштабируются по центральности.

После нахождения подходящего решения получился граф вида "лассо" (кольцо из вершин 0-25, далее прямая цепочка вершин от 25 до 49)

## Лаба 3. Модель Эрдёша–Реньи

### Условие

Сгенерировать граф `G(n, p)` с `n = 60`, `p = 0.45`. Посчитать эмпирическую среднюю степень и сравнить с теоретическим значением из лекций.

### Теория

| Величина                            | Формула                                                    | При наших параметрах |
| ------------------------------------------- | ----------------------------------------------------------------- | -------------------------------------- |
| Среднее число рёбер        | `m = n(n-1)/2 · p`                                             | `796.5`                              |
| Средняя степень               | `c = (n-1) · p`                                                | `26.55`                              |
| Распределение степеней | стремится к Пуассону `P(x) = c^x e^(-c) / x!` |                                        |

### Экспериментальное вычисление

Средняя степень считается как  `sum(d) / n` 

```python
m_emp = G.number_of_edges()
degrees = [d for _, d in G.degree()]
avg_degree_emp = sum(degrees) / n 


```

Дополнительно прогоняется 500 независимых генераций, средние степени по выборкам усредняются - должно почти точно сходиться к теоретическому значению.

### Пример вывода

```
Параметры: n=60, p=0.45

Рёбер (эмпирически):     771
Рёбер (теоретически):    796.50

Средняя степень эмпир.:  25.7000
Средняя степень теор.:   26.5500   = (n-1) * p
Расхождение:             0.8500

Среднее по 500 генерациям: 26.5719
Теория:                    26.5500
Std по генерациям:         0.6941
```

### Результаты

На одной реализации отклонение от теоретического значения ожидаемо - каждая выборка случайна. При усреднении по большому числу генераций расхождение стремится к нулю (закон больших чисел), что и наблюдается: 26.57 ≈ 26.55.

Гистограмма распределения степеней (по всем 500 графам) сравнивается с пуассоновским распределением с параметром `c = (n-1)·p`. Качественно похоже, но не идеально - формальный предельный переход от биноминального распредления к распредедению к Пуассоновскому работает при `n → ∞, np = const`, чего в данной задаче нет
